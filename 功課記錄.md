> 本檔記錄本專案之架構、各 API 分工、部署做法與曾遇到之問題與解法，方便日後維護與交功課／參賽說明。  
> **切勿**將真實 API Key、密碼、Telegram token 貼入版本庫；以下一律用佔位符表示。

## 一、專案係乜

- **名稱：** TON TE AI（Telegram Bot）
- **定位：** TON 區塊鏈相關查詢與操作（餘額、交易、Jetton、NFT、DNS、Swap 報價、發送 TON 等），用自然語言（文字 + 語音）互動。
- **參賽脈絡：** Identity Hub AI Hackathon（User-Facing Agents 類）等展示用途。
- **主要程式：** `bot.py`（Telegram + STT/TTS）、`ai.py`（DeepSeek + 工具呼叫）、`ton_tools.py`（鏈上 API）、`mcp_client.py`（可選 @ton/mcp）。

---

## 二、三層「智能」分工（一定要識）

| 層次 | 做咩 | 用邊個技術 / API |
|------|------|------------------|
| **聽（語音→文字）** | 將用戶語音變成文字 | **Groq**（Whisper API，`whisper-large-v3-turbo`）優先；無 key 時 **faster-whisper**（本地，預設 `small`）；再不行 **Google Web Speech**（後備） |
| **諗（推理+工具）** | 理解意圖、呼叫 TON 工具、生成回覆 | **DeepSeek**（`ai.py`，function calling） |
| **講（文字→語音）** | Bot 語音回覆 | **唔係 Groq**：廣東話優先 **cantonese.ai**（有 `CANTONESE_TTS_API_KEY` 時）；否則 **Microsoft Edge TTS**（`edge_tts`） |

**一句總結：** Groq = 只負責「聽」；DeepSeek = 「大腦」；Edge / cantonese.ai = 「讀出嚟」。

## 三、環境變數（`.env`）要記低嘅項目

```env
# 必填
TELEGRAM_BOT_TOKEN=        # BotFather
DEEPSEEK_API_KEY=          # DeepSeek 對話與工具

# 強烈建議（語音識別質素）
GROQ_API_KEY=              # Groq Cloud，格式 gsk_...

# 可選
WHISPER_MODEL=small        # 無 Groq 時本機 faster-whisper 模型：base | small | medium（視 VPS 記憶體）
CANTONESE_TTS_API_KEY=     # 廣東話 TTS 優先
TONCENTER_API_KEY=
MCP_SERVER_URL=
```

**注意：** 真實 key 只放伺服器 `.env`，唔 commit。

## 四、語音識別（STT）實際流程

1. 用戶發語音 → Telegram 下載音檔（多為 OGG）。
2. `pydub` 轉成 WAV。
3. **Tier 1：** 若有 `GROQ_API_KEY`，呼叫 Groq Whisper（語言對應見 `_GROQ_LANG_MAP`，廣東話 `yue`）。
4. **Tier 2：** 失敗則 **faster-whisper**（`WHISPER_MODEL`，預設 `small`；廣東話用 `initial_prompt` + 可自動偵測語言）。
5. **Tier 3：** Google `speech_recognition`（單語，後備）。
6. 可選後處理 `_fix_crypto_terms`（修正 TON/USDT 等聽錯字，**唔好**亂改粵語助詞）。

## 五、部署（LA VPS）

- **慣例：** Bot 跑喺 **LA VPS**（用戶指定之部署機）；**唔好**喺本地 Mac 跑帶 Telegram session 嘅其他腳本（見專案 `.cursorrules` 單一 IP 規則）。
- **服務：** `systemd` 例如 `tonpal.service`，主程式 `python3 /root/tonpal/bot.py`（路徑以實際為準）。
- **更新程式：** `scp` 上傳 `bot.py` 等 → `systemctl restart tonpal` → `journalctl -u tonpal -n 50` 睇 log。
- **資產：** `assets/banner.png`（或 webp）需存在，否則 `/start` 無圖。

## 六、Groq API Key 點樣攞（實戰經驗）

1. **地區：** 若 console 被擋，需 **VPN**（例如 Hiddify 掃 QR 導入）連到可訪問地區。
2. **登入方式建議：**
   - 優先 **Continue with email**：唔使 Google「通行密匙」；收到信後 **盡快、喺同一部機瀏覽器** 撳 link（link **一次性**，過期或重複用會報錯）。
   - **Continue with Google**：只係用 Google 帳號登入 Groq，**唔係** Google Cloud API Key；若帳號強制 Passkey，可改 email 登入。
3. **攞 Key：** 登入後左邊 **API Keys** → **Create API Key** → 複製 **`gsk_` 開頭**。
4. **寫入伺服器：** 在 VPS `/root/tonpal/.env` 設 `GROQ_API_KEY=gsk_...`，然後 `systemctl restart tonpal`。
5. **自動化限制：** 喺低配 VPS 用無頭瀏覽器幫你撳 Stytch magic link **唔可靠**（頁面重、token 易過期）；**最穩係人手喺自己瀏覽器完成一次登入並複製 key**。

### 備用 Groq Key

- **本機專案目錄（已加入 `.gitignore`，唔會 push 上 Git）：** `tonpal/.groq_backup`  
  - 內容為備用 `GROQ_API_KEY_BACKUP=gsk_...`，主 key 仍用 `.env` 嘅 `GROQ_API_KEY=`。
- **LA VPS：** `/root/tonpal/.groq_backup`（權限 `600`），與上面同一備用 key。
- **切換備用：** SSH 上 VPS，將備用 key 複製到 `/root/tonpal/.env` 嘅 `GROQ_API_KEY=` 一行（取代舊值），然後 `systemctl restart tonpal`。

## 七、UI／體驗曾做過嘅重點

- **歡迎／說明文字：** 可用 Telegram **自訂 emoji**（`<tg-emoji emoji-id="...">`）於**訊息內文／圖片說明**；**inline 按鈕文字**唔支援該語法，要用一般 Unicode emoji，否則會亂碼。
- **官方 TON 圖示：** 自訂 emoji 來自 Telegram **TONEmoji** 等 sticker set，與「假鑽石圖」區分。
- **Banner：** `/start` 刪舊 banner、發新圖，避免畫面堆疊。
- **語音：** 用戶語音可刪除、主畫面用 banner **原地編輯**；Bot 語音回覆可自動刪除（延遲秒數見程式）。
